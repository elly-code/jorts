
# UI choices

## On Linux

### Why no remember window positions

Because the toolkit does not allow it. So the app cannot control any window position.

elementary OS remembers window positions, but any others you'd have to ask why they dont.



### Why no window buttons

If you do not need a note anyore, then you delete it

If a note is currently not relevant, then it is covered up by another window, application

I do not see the point of maximize. Regardless, each note is maximizable if you snap it against borders on any modern WM


### Why no bold, italic, title, etc

if people need formatting, then they need libreoffice, not sticky notes

Exception is made for bullet lists because notes tend to have lists in them



### Why whimsy seasonal variants

whimsy



### Why random titles in new notes

showcase that notes can have custom editable titles

less boring than the apps name

whimsy



### Why monospace/normal font choice

some people have an easier time reading monospaced fonts

and you can also do ascii art on notes this way


### Why force elementary OS stylesheet

To avoid unpredictable weirdness on whatever system the app gets installed on

Any attempt at not doing that resulted in a broken weird looking app, in the best of cases

Everything should be exactly like the screenshot

### Why can autostart be desynchronized with the system

Jorts relies on Libportal for autostart. Libportal cannot tell it if it was set or not

So if any other tools is used to change autostart apps and add or remove this one, it is not aware

It only remembers what the user asked it directly
